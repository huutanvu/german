# German Learning Vault: Grist-Only UI and Architecture Plan

This document outlines the architecture, database schemas, and workflows for building a web-based user interface and local CLI backend for the German Learning Vault using Grist as the single database and backend.

---

## 1. System Architecture

```
┌────────────────────────────────────────────────────────┐
│                   Next.js Frontend                     │
│  - Personal/Family Dashboard (Dark Mode UI)            │
│  - Vocab review UI, Writing, Reading & Speaking pages  │
│  - Direct APIs to Grist (Data) & Publitio (Audio)      │
└───────────┬────────────────────────────────────────────┘
            │                                 ▲
            │ 1. Writes user responses/audio   │ 4. Reads corrections,
            │    and sets pending status      │    TTS audio & updates
            ▼                                 │
┌────────────────────────┐        ┌───────────┴──────────┐
│        Publitio        │        │        Grist         │
│  (Cloud Audio Files)   │        │   (Database/Backend) │
└────────────────────────┘        └──────────────────────┘
            ▲                                 ▲
            │ 3. Uploads target TTS           │ 2. Pulls pending work,
            │    audios & updates Grist       │    runs LLM, and
            │                                 │    writes corrections
            └───────────┬─────────────────────┘
┌───────────────────────┴────────────────────────────────┐
│             Offline Antigravity CLI (AGY)              │
│  - Local agent workflow using MCP server tools         │
│  - Pulls pending records directly from Grist           │
│  - Performs grading, correction, and TTS generation    │
│  - Writes results and media references back to Grist   │
└────────────────────────────────────────────────────────┘
```

- **Next.js Frontend**: Serves as the UI for user and family members. Connects directly to Grist for data storage and Publitio for media. Employs Supabase for authentication.
- **Grist**: Serves as the database. Stores all vocabulary, review attempts, writing exercises, reading passages, speaking files, and learning context.
- **Publitio**: Handles hosting of user-recorded audio (speaking practice) and system-generated audio (reading text TTS, target pronunciation TTS).
- **Offline CLI (AGY)**: Run locally by the user. Connects directly to Grist via MCP tools, processes pending reviews, writing paragraphs, reading comprehension, and speaking recordings, then writes the results back to Grist.

## 2. Grist Database Schema Design

### Table: `Vocabulary`
Holds the user's vocabulary library.
- `id` (Integer, Auto-increment)
- `word` (Text, Unique): German word/phrase
- `meanings` (Text): English meanings/translations
- `level` (Choice: A1, A2, B1, B2, C1, C2)
- `type` (Choice: new, revised, permanent, complicated)
- `correctCount` (Integer): Incremented on correct reviews (reaches permanent at 5)
- `grammar` (Text): Singular, plural, gender (nouns), auxiliary verb (verbs), etc.
- `dailyUse` (Text): Example sentence for daily usage
- `professionalUse` (Text): Example sentence in a software engineering context
- `tips` (Text): Correct prepositions/cases
- `caution` (Text): Common mistakes
- `updatedAt` (DateTime)

### Table: `VocabularyReviews`
Tracks the history and status of review sessions.
- `id` (Integer, Auto-increment)
- `vocabId` (Reference to `Vocabulary`)
- `userSentence` (Text): Sentence drafted by the user in the UI
- `correctedSentence` (Text): Corrected sentence generated by the CLI agent
- `correctionFeedback` (Text): Detailed grammar, orthography, and word choice feedback
- `status` (Choice: pending_correction, corrected, failed)
- `reviewedAt` (DateTime)

### Table: `WritingPractice`
Tracks writing practice sessions.
- `id` (Integer, Auto-increment)
- `topic` (Text): Topic generated by the CLI agent based on context
- `description` (Text): Guidance/instructions for the writing prompt
- `userParagraph` (Text): Paragraph written by the user
- `correctedParagraph` (Text): Fully corrected paragraph from the CLI agent
- `correctionsJson` (Text): Sentence-by-sentence breakdown of errors (grammar, spelling, word choice)
- `status` (Choice: pending_user, pending_correction, corrected)
- `date` (Text: YYYY-MM-DD)

### Table: `ReadingPractice`
Tracks reading comprehension sessions.
- `id` (Integer, Auto-increment)
- `topic` (Text): Article/reading topic
- `germanText` (Text): Reading paragraphs (250-350 words)
- `audioFileId` (Text): Publitio file ID of the TTS readout audio
- `questionsJson` (Text): 5 comprehension questions generated by the CLI agent
- `userAnswersJson` (Text): User's answers to the questions
- `correctionsJson` (Text): Evaluations and grammar corrections for user answers
- `status` (Choice: pending_user, pending_evaluation, evaluated)
- `date` (Text: YYYY-MM-DD)

### Table: `SpeakingPractice`
Tracks speaking and pronunciation practice sessions.
- `id` (Integer, Auto-increment)
- `topic` (Text): Speaking task/topic
- `targetText` (Text): Reference text to read out loud or question to answer
- `userAudioFileId` (Text): Publitio file ID of the user's recorded audio
- `transcript` (Text): Speech-to-text transcript generated by the CLI agent
- `grammarFeedback` (Text): Grammatical corrections of spoken sentences
- `pronunciationFeedback` (Text): Words mispronounced, with phonetic corrections
- `targetAudioFileId` (Text): Publitio file ID of the TTS target audio
- `score` (Integer): Pronunciation/fluency score out of 100
- `status` (Choice: pending_recording, pending_assessment, assessed)
- `date` (Text: YYYY-MM-DD)

### Table: `LearningContext`
Tracks the global state of the learning sessions.
- `id` (Integer, Auto-increment)
- `targetLevel` (Choice: A1, A2, B1, B2, C1, C2)
- `currentTopic` (Text): Active software engineering or daily topic
- `professionalEnvironment` (Text): Target working context (e.g. Software Engineer)
- `updatedAt` (DateTime)

## 3. UI Component Structure (Next.js)

### Pages
- **`/dashboard`**: Master dashboard showing daily learning progress, current level, active topic, and links to practice modules. All data is fetched from the Grist tables.
- **`/vocabulary`**: Renders "Today's New Words" using Grist `Vocabulary` table. Shows sentence-writing input boxes and logs them directly to `VocabularyReviews`.
- **`/writing`**: Reads topics from `WritingPractice` and allows typing paragraphs, submitting directly to Grist.
- **`/reading`**: Reads `ReadingPractice` text, streams TTS audio from Publitio CDN, and collects question answers to submit to Grist.
- **`/speaking`**: Displays reference text, records user's voice using browser APIs, uploads files directly to Publitio, and posts references to `SpeakingPractice` in Grist.

## 4. Offline Processing Workflow (CLI Agent via MCP)

The local CLI agent script connects directly to Grist using the MCP server.

### Stage 1: Check Grist Queue
The CLI agent queries Grist for rows with pending statuses:
- `VocabularyReviews` -> `status == "pending_correction"`
- `WritingPractice` -> `status == "pending_correction"`
- `ReadingPractice` -> `status == "pending_evaluation"`
- `SpeakingPractice` -> `status == "pending_assessment"`

### Stage 2: Execute AI Skills and Update Grist
1. **Vocabulary reviews**: Corrects sentences, calculates updated `correctCount`, patches `VocabularyReviews` status, and updates `Vocabulary`.
2. **Writing**: Grades the paragraph, produces sentence-by-sentence JSON feedback, updates `WritingPractice`.
3. **Reading**: Generates readings/comprehension questions when requested, writes target TTS audio files, uploads them to Publitio, and inserts the rows. Grades pending answers.
4. **Speaking**: Transcribes user audio files from Publitio URLs, analyzes pronunciation/grammar, generates target TTS, uploads it to Publitio, and updates Grist.
