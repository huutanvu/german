# Project Rules: German Learning Vault (Grist-Only Backend)

This folder `/mnt/d/german/` is the workspace for the German Learning Application. All data, texts, vocabularies, writing practices, and learning contexts are stored in Grist. AI agents interact with the data exclusively through the MCP server tools. Do not read or write markdown files on the local filesystem.

---

## 1. Data Model (Grist Tables)

### Table 1: `LearningContext`
Stores the global state of the learning sessions.
- `id` (Integer): Auto-incremented row ID
- `targetLevel` (Choice: A1, A2, B1, B2, C1, C2)
- `currentTopic` (Text): Target topic (e.g., General Software Engineering, Code Review)
- `professionalEnvironment` (Text): Default is Software Engineer
- `updatedAt` (DateTime)

### Table 2: `Vocabulary`
Holds the user's vocabulary list.
- `id` (Integer): Auto-incremented row ID
- `word` (Text): German word/phrase (AI will resolve raw captures to infinitives/nominative forms)
- `meanings` (Text): English meanings/translations
- `level` (Choice: A1, A2, B1, B2, C1, C2)
- `type` (Choice: new, revised, permanent, complicated)
- `correctCount` (Integer): Correct review count (mastery at 5)
- `grammar` (Text): Article, plural, genitive (nouns), auxiliary verb (verbs), etc.
- `dailyUse` (Text): Daily example sentence
- `professionalUse` (Text): Professional example sentence (Software Engineering context)
- `tips` (Text): Associated prepositions/cases
- `caution` (Text): Common pitfalls or false friends
- `context` (Text): Sentence context where this word was captured in the UI
- `isProcessed` (Bool): Flag indicating if this word has been processed and populated by the AI
- `updatedAt` (DateTime)

### Table 3: `VocabularyReviews`
Tracks individual review attempts and user sentence corrections.
- `id` (Integer): Auto-incremented row ID
- `vocabId` (Reference to `Vocabulary` table)
- `userSentence` (Text): Sentence written by the user in the UI
- `correctedSentence` (Text): Corrected sentence generated by the AI
- `correctionFeedback` (Text): Grammatical, orthographical, and lexical feedback
- `status` (Choice: pending_correction, corrected, failed)
- `reviewedAt` (DateTime)

### Table 4: `WritingPractice`
Tracks writing practice sessions.
- `id` (Integer): Auto-incremented row ID
- `topic` (Text): Topic generated by the AI based on context
- `description` (Text): Instructions/guidelines for the writing prompt
- `userParagraph` (Text): Paragraph written by the user
- `correctedParagraph` (Text): Corrected paragraph from the AI
- `correctionsJson` (Text): Sentence-by-sentence analysis of errors
- `status` (Choice: pending_user, pending_correction, corrected)
- `date` (Text: YYYY-MM-DD)

### Table 5: `ReadingPractice`
Tracks reading comprehension sessions.
- `id` (Integer): Auto-incremented row ID
- `topic` (Text): Reading topic
- `germanText` (Text): Paragraphs in German (250-350 words)
- `audioFileId` (Text): Publitio file ID of the read-out-loud audio
- `questionsJson` (Text): 5 comprehension questions generated by the AI
- `userAnswersJson` (Text): User's answers
- `correctionsJson` (Text): Evaluation and grammar corrections for user answers (structured JSON array format)
- `status` (Choice: pending_user, pending_evaluation, evaluated)
- `date` (Text: YYYY-MM-DD)

### Table 6: `SpeakingPractice`
Tracks speaking and pronunciation sessions.
- `id` (Integer): Auto-incremented row ID
- `topic` (Text): Speaking task/topic
- `targetText` (Text): Reference text to read out loud or question to answer
- `userAudioFileId` (Text): Publitio file ID of the user's recorded audio
- `transcript` (Text): Transcribed speech generated by the AI
- `grammarFeedback` (Text): Grammatical corrections
- `pronunciationFeedback` (Text): Mispronounced words and corrections
- `targetAudioFileId` (Text): Publitio file ID of target pronunciation TTS audio
- `score` (Integer): Pronunciation score out of 100
- `status` (Choice: pending_recording, pending_assessment, assessed)
- `date` (Text: YYYY-MM-DD)

## 2. MCP Server Tools

AI agents must use the following tools to interact with Grist:

### Context Tools
- `get_learning_context`: Returns target level and current topic.
- `update_learning_context`: Updates target level and current topic.

### Vocabulary Tools
- `list_vocabulary`: Enumerate vocabulary items, optionally filtered by `level` or `type`.
- `add_vocabulary`: Create a new entry in `Vocabulary` table.
- `update_vocabulary`: Edit fields of a vocabulary item (supports setting `isProcessed` and `context`).
- `list_reviews`: Fetch records from `VocabularyReviews`, filtered by `status`.
- `update_review`: Write corrections and feedback to a review record.

### Writing Tools
- `list_writing_practice`: Enumerate sessions, filtered by `status`.
- `upsert_writing_practice`: Create or update a writing practice session.

### Reading Tools
- `list_reading_practice`: Enumerate sessions, filtered by `status`.
- `upsert_reading_practice`: Create or update a reading practice session.

### Speaking Tools
- `list_speaking_practice`: Enumerate sessions, filtered by `status`.
- `upsert_speaking_practice`: Create or update a speaking practice session.

## 3. AI Instructor Workflows

When executing a correction or generation task:

1. **Daily Review Processing**:
   - Query `list_reviews` where `status = "pending_correction"`.
   - For each review: check the user's sentence, provide spelling/grammar/style feedback, determine if correct, increment or reset `correctCount` in `Vocabulary`, and call `update_review` to save results and change status to `corrected` or `failed`.
2. **Vocabulary Capture & Stemming Processing**:
   - Query `Vocabulary` records where `isProcessed = false`.
   - For each record:
     - Read the raw clicked `word` and the full sentence `context` where it was clicked.
     - Analyze the context sentence to resolve the correct infinitive (handling German separable verbs, noun capitalization, and stem mapping).
     - Update the Grist record's `word` field to this resolved infinitive form.
     - Generate meanings, grammar notes, daily and professional example sentences, tips, and cautions.
     - Set `isProcessed` to `true` and update the Grist record.
3. **Writing Corrections**:
   - Query `list_writing_practice` where `status = "pending_correction"`.
   - Perform sentence-by-sentence analysis (Grammar, Orthography, Lexicon).
   - Call `upsert_writing_practice` with the corrections and set status to `corrected`.
4. **Reading Evaluation**:
   - Query `list_reading_practice` where `status = "pending_evaluation"`.
   - Evaluate the 5 user answers. Format the evaluation feedback as a structured JSON array string matching the questions index (containing keys `question`, `userAnswer`, `evaluation`, `correction`, and `explanation`).
   - Write it to Grist via `upsert_reading_practice` and set status to `evaluated`.
5. **Speaking Assessment**:
   - Query `list_speaking_practice` where `status = "pending_assessment"`.
   - Retrieve user audio file URL (Publitio), transcribe it, analyze pronunciation accuracy, run TTS helper to generate correct pronunciation audio, upload to Publitio, and call `upsert_speaking_practice` to write results and set status to `assessed`.

## 4. Mobile-First Application Architecture

The application is structured to be mobile-first and responsive:
- **Navigation Model**: Dashboard lists (e.g. `/reading`, `/writing`, `/speaking`) present search, sorting, and status filters in a clean grid.
- **Dynamic Route Detail Pane**: Clicking an item in a list routes directly to its corresponding dynamic route (`/reading/[id]`, `/writing/[id]`, `/speaking/[id]`). This allows full-width focus suitable for mobile screens and shareable URLs.
- **Interactive Reading**: Text passages split into sentence and word tokens. Clicking a word opens a responsive `@radix-ui/react-popover` containing inline links to lookup definitions or add the word to Grist.
- **Side Drawer Info**: Sidebar lookup panel draws from the right to display full definitions and grammar contexts.

## 5. Note Writing Styles
- **No Emojis or Icons**: Do not use emojis, icons, or emoticons.
- **Formatting Tools**: Use bold, quotes, code blocks, and italics for clean structure.
- **Cleanliness**: Keep notes short, clean, bulleted, and focused.

---

## 6. Interaction Rules & Output Constraints
- **Clarification Over Guessing**: Ask if unsure.
- **Concise Chat Responses**: Keep chat responses extremely short and clean. Only output a brief pointer of what note was created or updated.
